# Contributing

Thanks for your interest in contributing.

## Development Flow
1. Fork or create a feature branch from `main`.
2. Keep changes small and focused.
3. Add/update tests when behavior changes.
4. Open a pull request with context, rationale, and validation steps.

## Pull Request Checklist
- [ ] Code builds successfully
- [ ] Tests pass locally
- [ ] Documentation updated (if needed)
- [ ] No sensitive data or secrets committed

## Code Standards
- Prefer readability and maintainability over cleverness
- Follow existing project structure and naming patterns
- Include concise comments only where necessary
